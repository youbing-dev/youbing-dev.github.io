问题现象

最终结果

经验总结

  1. 新表上线时一定要评估查询模式，提前创建索引
  2. EXPLAIN ANALYZE 是排查慢查询的第一工具
  3. 覆盖索引（INCLUDE）可以避免回表，效果显著
  4. LEFT JOIN 和 INNER JOIN 的执行计划差异巨大，按需选择
